## 二、后端开发

### 3. 数据库设计提示词
```
设计图书管理系统的数据库表结构，包括：
1. users表 - 用户信息（id, username, password, name, email, phone, role, created_at）
2. books表 - 图书信息（id, isbn, title, author, publisher, publish_date, category_id, description, total_copies, available_copies, location, cover_image, created_at）

```

### 4. API接口开发提示词
```
开发以下RESTful API接口：

用户认证API：
- POST /api/auth/register - 用户注册
- POST /api/auth/login - 用户登录
- GET /api/auth/profile - 获取个人信息
- PUT /api/auth/profile - 更新个人信息
- PUT /api/auth/password - 修改密码

图书管理API：
- GET /api/books - 获取图书列表（支持分页、搜索、筛选）
- GET /api/books/:id - 获取图书详情
- POST /api/books - 添加图书（管理员）
- PUT /api/books/:id - 更新图书（管理员）
- DELETE /api/books/:id - 删除图书（管理员）
- POST /api/books/:id/cover - 上传封面（管理员）

借阅管理API：
- GET /api/borrow - 获取借阅记录
- POST /api/borrow/borrow/:bookId - 借阅图书
- POST /api/borrow/return/:borrowId - 归还图书
- POST /api/borrow/renew/:borrowId - 续借图书

分类管理API：
- GET /api/categories - 获取分类列表
- POST /api/categories - 添加分类（管理员）
- PUT /api/categories/:id - 更新分类（管理员）
- DELETE /api/categories/:id - 删除分类（管理员）

统计分析API：
- GET /api/statistics/overview - 获取概览统计
- GET /api/statistics/category - 获取分类统计
- GET /api/statistics/popular-books - 获取热门图书
- GET /api/statistics/active-users - 获取活跃用户
- GET /api/statistics/borrow-trend - 获取借阅趋势
- GET /api/statistics/overdue - 获取逾期列表

用户管理API（管理员）：
- GET /api/users - 获取用户列表
- PUT /api/users/:id/role - 更新用户角色
- DELETE /api/users/:id - 删除用户

要求：
1. 使用JWT进行身份认证
2. 管理员接口需要验证admin权限
3. 返回统一的JSON格式 { success, data/error, pagination }
4. 借阅规则：借期30天，可续借一次，逾期每天罚款0.5元
```

## 四、UI主题设计

### 7. 星空主题设计提示词
```
使用ui-ux-pro-max技能对前端界面进行全面升级，将其视觉风格改造为"星空图书馆"主题。

设计要求：
1. 简约美学原则，以白色星点作为核心视觉元素
2. 星点大小、密度和分布应呈现自然随机的星空效果
3. 严格禁止使用任何紫色调
4. 整体配色方案以中性色调为基础，可适当搭配深蓝或深灰作为星空背景色
5. 界面升级需保持功能完整性和交互流畅性
6. 确保响应式设计适配不同设备屏幕尺寸
7. 优化加载性能，避免因星空效果导致页面加载延迟或运行卡顿

具体实现：
- 背景色：深蓝黑色渐变（#0a0f1a → #1a1f2e → #0d1220）
- 星点效果：三层大小不同的白色星点 + 闪烁动画 + 流星效果
- 主色调：蓝色（#3b82f6）
- 卡片样式：毛玻璃效果（backdrop-filter: blur）+ 半透明背景
- 文字层级：主文字 #f1f5f9 / 次要 #94a3b8 / 弱化 #64748b
```
